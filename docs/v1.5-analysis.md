# supportutils-scrub v1.5 Release — Phase 1 Analysis

Scope: read-only audit of the repository at current `main` head (1 commit ahead of
origin) with working-tree change in `cli.py`. Goal is to scope the v1.5 work:
what to fix, what to refactor, and what to leave alone.

No code has been changed yet. Review this file and pick what should land in
v1.5 before Phase 2 begins.

---

## 1. Repo map

**Version markers**
- `src/supportutils_scrub/main.py:3-4` — `SCRIPT_VERSION = "1.4"`, `SCRIPT_DATE = "2026-04-01"`
- Exit codes declared in the same file: `EXIT_OK=0`, `EXIT_ERROR=1`, `EXIT_WARNING=2`, `EXIT_VERIFY_FAIL=3`

**Entry points**
- `bin/supportutils-scrub` — shim that hard-codes `/usr/lib/supportutils-scrub/` on `sys.path` then calls `main.main`
- `setup.py` — `console_scripts: supportutils-scrub=supportutils_scrub.main:main`
- `main.py` — delegates to `cli.main`
- `cli.py` — argparse wiring and mode dispatch

**Modes** (`src/supportutils_scrub/modes/`)
| File | Responsibility |
|---|---|
| `archive.py` | `.txz` / `.tgz` / `.tar.gz` archives, multi-archive with shared mappings |
| `folder.py` | existing supportconfig directory, produces `<input>_scrubbed/` |
| `file.py` | single plain text file, output `<input>_scrubbed` |
| `stdin.py` | stdin/stdout, bulk or `--stream` line-buffered |

**Scrubbers** (11 modules, all subclass `Scrubber` in `scrubber.py`)
`ip`, `ipv6`, `mac`, `domain`, `hostname`, `user`, `keyword`, `serial`,
`email`, `password`, `cloud_token`

**Shared pipeline** (`pipeline.py`) — domain/hostname/username/serial extraction,
subnet-aware pool helpers, supportconfig folder detection, hostname-based rename.

**Verify** (`verify.py`) — 5 layers: mapping-based, IPv4 allowlist,
MAC allowlist, pattern scan (emails/secrets/LDAP/Kerberos), identity extraction
from original.

**Audit / reports** (`audit.py`, `translator.py`) — mapping save/load (plain
and Fernet-encrypted), SHA256, JSON report, JSON audit log.

**Config** — INI-style file parsed by `config_reader.py` into
`scrub_config.ScrubConfig` dataclass. Default path
`/etc/supportutils-scrub/supportutils-scrub.conf`.

**Packaging / man pages**
- `config/supportutils-scrub.conf` — shipped default
- `man/supportutils-scrub.8` (20 K) and `man/supportutils-scrub.conf.5` (4.5 K)
- `Makefile` — only `test`, `test-verbose`, `test-quick`, `lint` targets
- `setup.py` uses absolute `data_files` paths (`/etc/...`, `/usr/share/man/...`)
- **No `.spec` file**, **no `CHANGELOG.md`**

**Tests** — 17 files, ~1700 lines. Fixtures at `tests/fixtures/`.
Contract, unit, integration, and network-integrity layers. Per saved
preference, end-to-end runs are left to the user.

**Version-gated logic in the companion `supportconfig` bash integration**
(outside this repo) expects: exit codes 0/1/2/3 present since v1.3;
stdout path of scrubbed dir present since v1.4; hostname folder rename
since v1.4; `--quiet` since v1.3.

## 2. Findings

Severity scale: **HIGH** = correctness / leaks / release blocker,
**MED** = behaviour gap the user already hit,
**LOW** = ergonomics / cleanup.

### 2.1 Bugs

**B1 — IPv6 addresses ending in `::` are not matched. [HIGH]**
- `src/supportutils_scrub/ipv6_scrubber.py:9`
- `CANDIDATE_V6` ends `[0-9A-Fa-f:.]+[0-9A-Fa-f]`, forcing the last char to be
  a hex digit. Real tokens like `2600:1901:0:7018::` end in `:` and are never
  scanned.
- Fix sketch: allow `::` terminator explicitly, e.g.
  `([0-9A-Fa-f]+(?::[0-9A-Fa-f]*)*(?:::)?)` or append an `|::$` alternative.
  Requires regression test in `test_ipv6_scrubber.py`.

**B2 — IPv6 ULA and link-local are always skipped. [MED]**
- `src/supportutils_scrub/ipv6_scrubber.py:45-54` (`_skip_scope`)
- Returns `True` for `fc00::/7` and `fe80::/10` unconditionally. The IPv4 path
  honours `obfuscate_private_ip`; IPv6 ignores it, so ULA leaks through even
  when the user explicitly asks for private obfuscation.
- Fix sketch: gate ULA/link-local on `config.obfuscate_private_ip` (or
  introduce `obfuscate_private_ipv6` for symmetry). Add contract test.

**B3 — Verify's first-octet ≤ 2 blanket allowlist hides real leaks. [MED]**
- `src/supportutils_scrub/verify.py:84-94` (`_is_safe_ipv4`)
- Every `1.x.x.x` and `2.x.x.x` is declared safe to cut version-string noise.
  That swallows real addresses too (`1.1.1.1` Cloudflare DNS, `2.16.x.x` Akamai).
- Fix sketch: drop the blanket rule; instead reuse the same version-context
  lookbehind the scrubber already has at
  `src/supportutils_scrub/ip_scrubber.py:294-298`. Apply it to the verify IP
  allowlist check so `pkg-1.4.4.2`, `saptune-16.0.0.4`, and
  `version 2.25.1.0+git` are skipped contextually but `1.1.1.1` in
  `nameserver 1.1.1.1` is still flagged.

**B4 — `password_scrubber` misses colon-separated passwords. [MED]**
- `src/supportutils_scrub/password_scrubber.py:5-10`
- Regex requires `password\s*=`. YAML, `.netrc`, Ansible vault, and a lot of
  Kubernetes manifests use `password:`.
- Fix sketch: broaden the delimiter to `[:=]`, keep the `*REMOVED` /
  `scrubbed_pass_` negative lookaheads. Add regression test.

**B5 — `keyword_scrubber` generates non-deterministic, collision-prone replacements. [MED]**
- `src/supportutils_scrub/keyword_scrubber.py:39-41`
- `'x' * random.randint(7, 10)` means: (a) different mapping on every run even
  for identical inputs, (b) two keywords can collapse to the same `xxxxxxx`
  when their lengths collide, which makes the mapping lossy.
- Fix sketch: replace with `keyword_{idx}` (same pattern as other scrubbers).
  Ensures uniqueness and reproducibility. Mapping-file reuse via
  `--mappings` then actually works for keywords too.

**B6 — `audit.load_mappings_file` silently returns `{}` on corruption. [MED]**
- `src/supportutils_scrub/audit.py:61-66`
- Malformed JSON prints a warning and proceeds with an empty mapping. The
  caller thinks mappings were loaded and produces a fresh set of fakes,
  breaking the "consistent across runs" contract the user relies on.
- Fix sketch: `sys.exit(EXIT_ERROR)` when JSON decode fails (only on read
  errors — file-not-found can stay lenient if we want `--mappings newfile.json`
  to bootstrap).

**B7 — `modes/file.py` instantiates `ConfigReader` twice. [LOW]**
- `src/supportutils_scrub/modes/file.py:33-39`
- Two `ConfigReader(DEFAULT_CONFIG_PATH)` calls read the same file; the first
  result (`config_f`) is used only for `dataset_dir`, then we re-read.
- Fix sketch: drop the first block, reuse `args._preloaded_config` like
  `modes/folder.py` does. Same change applies to `modes/archive.py:203`
  and `modes/stdin.py:51`.

**B8 — `hostname_scrubber.build_hostname_dict` emits `hostname<n>`, pipeline emits `hostname_<n>`. [LOW]**
- `src/supportutils_scrub/hostname_scrubber.py:128` vs
  `src/supportutils_scrub/pipeline.py:104`
- Two different naming conventions. The static method looks unused now but
  the inconsistency is a trap; anything that reloads through it rebuilds
  mappings incorrectly.
- Fix sketch: align to `hostname_{n}` in both places or delete the static
  method if nothing calls it (verify with grep first).

**B9 — `cli.py` `--report` recovery is a hack. [LOW]**
- `src/supportutils_scrub/cli.py:108-114`
- Works, but papers over the `nargs='?'` ambiguity by sniffing the argument.
  Fragile against future positional additions.
- Fix sketch: split into boolean `--report` + explicit `--report-file FILE`,
  or drop `nargs='?'` and use a sentinel `const`. Update man page and
  `SUPPORTUTILS_SCRUB_OPTS` examples.

**B10 — `--rewrite-pcap` with archives runs both flows. [LOW]**
- `src/supportutils_scrub/modes/archive.py:207-227`
- No `return` after the pcap rewrite branch; control falls through and
  scrubs the archive too. May be intended; either way, undocumented.
- Fix sketch: decide semantics and either return or document in the man page.

**B11 — `extractor.py` top-level prefix detection is fragile. [LOW]**
- `src/supportutils_scrub/extractor.py:60-65` (both
  `extract_xz_archive` and `extract_tgz_archive`)
- Picks first non-empty member and treats its first path segment as the root.
  Breaks for archives with `./prefix` or mixed roots. Also duplicated across
  the two helpers.
- Fix sketch: use
  `os.path.commonpath([m.name for m in members if not m.issym() and m.name])`
  with a sentinel fallback. Unify both helpers behind one function
  parameterised by mode.

### 2.2 Robustness

**R1 — `_SAFE_IPV4_ALWAYS` pool range disagrees with `ScrubConfig.public_pool`. [MED]**
- `src/supportutils_scrub/verify.py:48` uses `198.16.0.0/12`
- `src/supportutils_scrub/scrub_config.py:26` uses `198.16.0.0/12` (same), but
  the CLAUDE.md doc and older comments mention `198.18.0.0/15`. Not a bug
  today but the comment drift is dangerous if the config gets changed.
- Fix sketch: either parametrise verify from the loaded config (preferred) or
  add a startup assertion that `verify._SAFE_IPV4_ALWAYS` contains
  `config.public_pool`. Also update CLAUDE.md.

**R2 — `_SAFE_IPV4_ALWAYS` is missing masks other than `255.0.0.0/8`. [LOW]**
- `src/supportutils_scrub/verify.py:46-65`
- `255.255.0.0`, `255.255.255.0`, `255.255.255.128` etc. are listed in
  `ip_scrubber.SPECIALS` but fall outside the verify safe list because the
  octet regex `_OCTET` treats them as normal IPs. Today they pass because
  `_is_safe_ipv4` falls through `0.x.x.x/127.x.x.x` etc., but only a few.
- Fix sketch: add `255.0.0.0/8` superset or check `ip_str in _SPECIALS`.

**R3 — `processor.py` detects binary by filename only. [LOW]**
- `src/supportutils_scrub/processor.py:34-44`
- Any file named `saYYYYMMDD` is deleted even if the file happens to be text.
  Unlikely in practice. Peek first 4096 bytes for NUL before deleting.

**R4 — Extractor duplication. [TECH DEBT]**
- `extract_xz_archive` and `extract_tgz_archive` are ~50 lines each with
  near-identical bodies. Fold into one helper taking `mode="r:xz"` /
  `"r:gz"`.

**R5 — Summary-block printing duplicated across modes. [TECH DEBT]**
- `modes/archive.py:306-345`, `modes/folder.py:174-204`,
  `modes/file.py:125-153`, `modes/stdin.py:154-181` all print very similar
  obfuscation summaries with small variations.
- Fix sketch: one helper in `audit.py` that takes a `dataset_dict` and
  writes the block to the chosen stream.

**R6 — Config re-read per mode. [LOW]**
- `cli.py` already preloads config into `args._preloaded_config` but only
  `folder.py` uses it. `archive.py:203`, `file.py:34-39`, `stdin.py:51`
  re-instantiate `ConfigReader`.
- Fix sketch: always use `args._preloaded_config`.

**R7 — `pipeline.extract_hostnames_from_hosts` opens files with no encoding / no exception handling. [LOW]**
- `src/supportutils_scrub/hostname_scrubber.py:39` — `open(file_path, 'r')`
- Breaks on non-UTF8 bytes in `/etc/hosts` dumps (rare but possible).
  Wrap with `encoding='utf-8', errors='ignore'` and try/except like
  `extract_usernames_from_section` already does.

**R8 — Verify identity check uses substring match. [LOW]**
- `src/supportutils_scrub/verify.py:587-595`
- `token_low in line_lower` triggers on any occurrence. A serial happening
  to appear inside a hex dump or a path yields a false-positive finding.
- Fix sketch: use word-boundary regex per token and require `len(token) ≥ 10`
  for identity.

**R9 — No cleanup of pcap `.tmp` file on tcprewrite failure. [LOW]**
- `src/supportutils_scrub/pcap_rewrite.py:97-107`
- `subprocess.run(... check=True)` raises before `os.replace(tmp, fout)`,
  leaving a partial `.tmp`.
- Fix sketch: `try/finally` removing `tmp` on exception.

### 2.3 Performance

**P1 — `keyword_scrubber.scrub` is O(n_keywords × len(text)). [MED]**
- `src/supportutils_scrub/keyword_scrubber.py:47-49`
- Runs `re.sub` once per keyword. With 50–100 keywords and multi-MB files
  this is measurable.
- Fix sketch: build one alternation `(?i)\b(kw1|kw2|...)\b`, dispatch to the
  fake value via a dict lookup in the replacer. Matches the approach in
  `HostnameScrubber` and `DomainScrubber`.

**P2 — `ip_scrubber` reconstructs `_sorted_subnets` on every call. [LOW]**
- `src/supportutils_scrub/ip_scrubber.py:189-191`, `201-204`
- Rebuilt inside `_prepare_subnets` and again in `_map_in_subnets` when
  invalidated. Fine per file but worth caching if profiling shows regression.

**P3 — Verify reopens every file. [LOW]**
- `src/supportutils_scrub/verify.py:490` reads each file line-by-line; that's
  fine. What's missing is a size cap. A 200 MB `messages.txt` will be scanned
  entirely with all regex layers. Acceptable but worth noting.

### 2.4 Packaging

**PK1 — No RPM spec file. [HIGH for v1.5 release]**
- `setup.py` builds an sdist/wheel but there is no `packaging/*.spec` to
  submit to OBS. This is a release blocker if the goal is to ship 1.5 to the
  openSUSE build service alongside supportutils.
- Fix sketch: add `packaging/supportutils-scrub.spec` with version from
  `main.py`, BuildArch `noarch`, Requires `python3`, installs binary to
  `/usr/bin/supportutils-scrub`, config to `/etc/supportutils-scrub/`,
  man pages to `%{_mandir}/man{5,8}`, and `%changelog` from CHANGELOG.md.

**PK2 — No `CHANGELOG.md`. [MED]**
- Version history lives only in git log + `SCRIPT_DATE`. Required for spec
  `%changelog` and for reviewers.
- Fix sketch: add `CHANGELOG.md` in Keep-A-Changelog format. Pre-populate
  1.2/1.3/1.4 entries from `git log --grep` on version-bump commits.

**PK3 — `bin/supportutils-scrub` duplicates the entry_point. [LOW]**
- `bin/supportutils-scrub:7` hardcodes `/usr/lib/supportutils-scrub/` on
  `sys.path`. The `setup.py` already declares
  `console_scripts: supportutils-scrub=supportutils_scrub.main:main`, which
  generates a working binary in `/usr/bin` at install time. The shim is
  redundant and fragile.
- Fix sketch: delete `bin/supportutils-scrub`, let pip/RPM install the
  console_script. If the `/usr/lib` path has to stay for legacy reasons,
  document why in the spec comment.

**PK4 — `setup.py` uses absolute `data_files` paths. [LOW]**
- `setup.py:32-36` writes to `/etc/` and `/usr/share/man/`, which makes
  wheels non-relocatable. Harmless under RPM but ugly for pip installs in
  a venv.
- Fix sketch: move installation of these files to the spec's `%install`
  section, drop them from `setup.py data_files`.

**PK5 — `Makefile` lacks `install`, `clean`, `dist`, `man` targets. [LOW]**
- Only test/lint targets exist.
- Fix sketch: add convenience targets so `make install` works
  (useful for RPM `%install` copy-in).

### 2.5 Tech debt

**T1 — Empty file `git` at repo root. [TRIVIAL]**
- `/home/ronald/Desktop/supportutils-scrub/git` — 0 bytes, almost certainly
  a shell-redirect typo (`> git status > git`). Ignored by `.gitignore`?
  It appears in `git status` as untracked. Delete.

**T2 — `keyword_scrubber` prints to stdout during `load_keywords`. [LOW]**
- `src/supportutils_scrub/keyword_scrubber.py:23`, `32`
- Prints `[✓] Loading keywords from file: ...` to stdout. In stdin/streaming
  mode this pollutes the scrubbed output pipe.
- Fix sketch: move to `logger.info` or `sys.stderr`.

**T3 — Two copies of the "scrubbed INFO header" constant. [LOW]**
- `src/supportutils_scrub/processor.py:51-55` and
  `src/supportutils_scrub/modes/file.py:87-91` define the same header block.
- Fix sketch: single constant in `scrubber.py` or `audit.py`.

**T4 — `config_reader.py` does not validate types. [LOW]**
- `scrub_config.from_dict` silently drops unknown keys. That is convenient,
  but typo'd booleans like `obfuscate_prvate_ip = yes` pass unnoticed.
- Fix sketch: emit a warning for unknown keys, keep the drop behaviour.

**T5 — `ip_scrubber.scrub_ip` legacy helper. [LOW]**
- `src/supportutils_scrub/ip_scrubber.py:319-340` — "Legacy single-IP scrub
  without subnet". Look for callers; delete if none.

**T6 — `_KERBEROS_RE` and `_EMAIL_RE` in `verify.py` are close to duplicates of
  the scrubber regexes. [LOW]**
- Same pattern ideas live in `email_scrubber.EMAIL_RE` and the verify module.
  Drift between them produces asymmetric findings.
- Fix sketch: import from the scrubber modules.

**T7 — `mac_scrubber._generate_fake_mac` overflows silently at 16,777,216 MACs. [LOW]**
- Counter is 3 bytes. Unlikely in one run, but reused mapping files that grow
  past the limit wrap around and collide.
- Fix sketch: use 5 bytes (extend past `00:1A:2B:` into a 2-byte region) or
  raise.

### 2.6 Docs / UX

**D1 — CLAUDE.md states "Private IPs are not obfuscated by default". [LOW]**
- `CLAUDE.md:83`. Current defaults: `ScrubConfig.obfuscate_private_ip=True`
  and `config/supportutils-scrub.conf` ships `obfuscate_private_ip = yes`.
- Fix sketch: update CLAUDE.md and the README section that mirrors it.

**D2 — `man/supportutils-scrub.8` does not document the v1.4 behaviours
  required by supportconfig integration. [MED]**
- Specifically the stdout path convention, `--quiet` contract, and exit codes
  2 / 3. These were introduced for `supportconfig -j`.
- Fix sketch: add an "INTEGRATION WITH supportconfig(8)" section.
- (The companion bash integration already lives in the supportutils repo,
  not here — but the contract belongs in our man page too.)

**D3 — No `CONTRIBUTING.md` / `SECURITY.md`. [LOW]**
- For a tool that handles PII, a `SECURITY.md` with responsible-disclosure
  contact is expected before an OBS / upstream push.

**D4 — Info header wording drift. [TRIVIAL]**
- `processor.py` header says "Review before sharing"; some comments and the
  README still reference an older wording. Pick one.

### 2.7 Test coverage gaps

- **TG1** — `verify.py` (623 lines, 5 layers) has zero unit tests.
- **TG2** — `cli.py` argument parsing is untested, including the `--report`

- **TG3** — No tests for any of the four modes (`archive`, `folder`, `file`,
  `stdin`). Coverage is implicit through `FileProcessor`.
- **TG4** — `extractor.py` (archive extraction, path traversal guard,
  `copy_folder_to_scrubbed`) and `pcap_rewrite.py` (tcprewrite argument

- **TG5** — No test for encrypted mappings

- **TG6** — No regression test for the signal-handler cleanup path.
- **TG7** — IPv6 tests cover a single `2001:db8::/32` case; no subnet-aware

- **TG9** — No corrupted-mapping-file regression (B6).
- **TG10** — No keyword performance / large-list test (P1).

## 4. Open questions before Phase 2

- **Version bump** — confirm `1.4` → `1.5`. Also bump `SCRIPT_DATE` and spec
  `%changelog`.
- **Private-IPv6 config key** — introduce a separate `obfuscate_private_ipv6`
  or reuse `obfuscate_private_ip` for both families? Reuse is simpler;
  separate is more flexible.
- **`--report` redesign** — split into `--report` + `--report-file` (breaking
  for scripted users) or keep the hack with a test? Recommend split with a
  one-line deprecation note in the man page.
- **PK3 bin shim** — delete or keep for `/usr/lib/supportutils-scrub/`
  back-compat? If RHEL-like distros ever pick this up, the shim helps.
